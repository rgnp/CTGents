# 开发路线图：终极目标 — 自进化 + AIGC

> 以当前 Agent 框架为基础，走向两个终极能力：
> **自进化（Self-Evolution）** — 给自己加想要的功能
> **AIGC（AI Generated Creativity）** — 产生创新的东西

---

## 🎯 愿景

```
当前 CTGents ───→ 自进化 Agent ───→ AIGC Agent
    │                    │                    │
    │ 你告诉我缺什么      │ 我自己发现缺什么    │ 我不但能改，还能创造
    │ 我手动改代码        │ 我自动加功能        │ 新的工具、新的交互
    └────────────────────┴────────────────────┴───────────────→
```

---

## 🚧 Phase 0：地基（当前状态）

已完工的基础能力：

| 能力 | 状态 | 说明 |
|------|:----:|------|
| 会话管理 | ✅ | 保存/恢复/重命名/导出/列表 |
| 指令系统 | ✅ | 结构化注册、热加载、/help 自动聚合 |
| MCP 协议 | ✅ | 连接任意 MCP 服务器，工具自动注册 |

| 工具去重 | ✅ | 同轮滑动窗口去重 |
| 安全模式 | ✅ | MANUAL / SAFE / AUTO 三级 |
| 去重拦截 | ✅ | 同工具连续相同参数自动拦截 |

## Phase 1：完善基础体验

> 补上当前最明显的短板，为自进化铺路。

### 1.1 自省能力

- [ ] Agent 能查看自己的完整工具列表、架构文档
- [ ] Agent 能读取自己的源代码
- [ ] Agent 能分析自己的性能瓶颈（慢在哪、错在哪）
- [ ] Agent 能统计自己的调用频率、失败率

### 1.2 用户画像记忆

- [ ] 持久化记忆系统（当前已有关键词记忆）
- [ ] 用户偏好自动学习（回答长度、输出风格）
- [ ] 项目级知识自动积累

### 1.3 执行稳定性

- [ ] 大任务自动分步执行（不依赖 /goal）
- [ ] 任务中途可中断、可续接
- [ ] 长时间任务进度提示

---

## Phase 2：自进化（Self-Evolution）

> Agent 能给自己加想要的功能。这是第一座里程碑。

### 2.1 插件热加载 ← 首步

- [ ] Agent 能通过 `install_plugin` 给自己装新能力
- [ ] Agent 能理解 Plugin 接口规范（`plugin_spec`）
- [ ] Agent 能自主编写插件代码并安装

### 2.2 自诊断

- [ ] Agent 发现工具调用高频失败 → 自动修复
- [ ] Agent 发现上下文超长 → 自动压缩策略调整
- [ ] Agent 发现重复劳动 → 自动合并优化

### 2.3 自扩展

- [ ] Agent 从新工具中学习使用方式（MCP 自动适配）
- [ ] Agent 遇到重复需求时自动封装为命令
- [ ] Agent 能改写自己的 prompt 策略

### 2.4 自进化闭环

```
发现问题 ─→ 分析根因 ─→ 生成方案 ─→ 执行修改 ─→ 验证 ─→ 记忆
   ↑                                                    │
   └────────────────────────────────────────────────────┘
```

- [ ] 闭环跑通：Agent 自主完成一次功能增减
- [ ] 安全护栏：修改前自动备份、自动测试
- [ ] 回滚机制：修改出问题自动回退

---

## Phase 3：AIGC（AI Generated Creativity）

> Agent 能产生创新的东西。这是第二座里程碑。

### 3.1 Agent 能创造新工具

- [ ] 根据自然语言需求编写新工具（如"加一个批量重命名工具"）
- [ ] 工具自动注册、自动生成测试
- [ ] 工具使用效果自动反馈优化

### 3.2 Agent 能创造新交互

- [ ] 发现重复模式时提议新命令 / 快捷键
- [ ] 自动生成交互式向导简化复杂操作

### 3.3 Agent 能创造新知识

- [ ] 从项目库中提炼最佳实践 → 生成文档
- [ ] 从多次修复中提炼模式 → 生成 check 规则
- [ ] 跨项目迁移经验（"这个 bug 在上个项目怎么修的"）

### 3.4 开放式创新

- [ ] "你有什么建议？"— 主动提改进方案
- [ ] 识别"可以做得更好"的地方
- [ ] 探索新技术并评估是否值得接入
