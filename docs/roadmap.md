# 开发路线图：终极目标 — 自进化 + AIGC

> 以当前 Agent 框架为基础，走向两个终极能力：
> **自进化（Self-Evolution）** — 给自己加想要的功能
> **AIGC（AI Generated Creativity）** — 产生创新的东西

---

## 🎯 愿景

```
我帮你做（现在）────→ 我给自己加功能（自进化）────→ 我创造新东西（AIGC）
                              ↑                               ↑
                     install_plugin 已能用             插件系统已就绪
                     缺的是"自主发现+闭环"             缺的是"主动性+创造力"
```

---

## 🚧 当前状态

### 已有能力

| 能力 | 状态 | 说人话 |
|------|:----:|--------|
| 会话管理 | ✅ | 保存/恢复/重命名/导出/列表 |
| 指令系统 | ✅ | 结构化注册、热加载、/help |
| MCP 协议 | ✅ | 连接任意 MCP 服务器 |
| 上下文缓存 | ✅ | 三段式 CacheContext |
| RAG 索引 | ✅ | 语义搜索项目代码 |
| 工具并行 | ✅ | 只读并行，写串行 |
| 工具去重 | ✅ | 同轮滑动窗口去重 |
| 安全模式 | ✅ | MANUAL / SAFE / AUTO |
| **插件热加载** | **✅** | **`install_plugin` 已可用，plugins/ 目录就绪** |

### 核心差距

**我缺的不是"能不能加功能"——`install_plugin` 已经能加。**
**我缺的是：**

1. **主动性** — 你不会说"帮我装个插件"，我先得等你告诉我需求
2. **自诊断** — 我不会主动发现"这个工具经常失败"或"这个流程可以自动化"
3. **闭环** — 改完不会自动验证 + 记忆

---

## Phase 1：自省 + 主动性（补齐差距）

> 让 Agent 能看见自己、分析自己、主动提议改进。

### 1.1 自省能力

- [ ] 查看自己的完整工具列表、架构文档
- [ ] 读取自己的源代码
- [ ] 分析调用频率、失败率、慢在哪
- [ ] 统计：哪些工具最常用、哪些最容易报错

### 1.2 主动提议

- [ ] 发现重复 3 次的操作 → 问"要不要封装成插件？"
- [ ] 发现工具高频失败 → 问"要不要修复？"
- [ ] 发现耗时操作 → 问"要不要优化？"
- [ ] 每次会话结束主动提 1 个改进建议

### 1.3 用户画像

- [ ] 持久化记忆系统（已有基础）
- [ ] 自动学习偏好（长度、风格）
- [ ] 项目级知识自动积累

---

## Phase 2：自进化闭环

> "发现问题 → 分析根因 → 生成方案 → 执行修改 → 验证 → 记忆"
> 整个闭环自主完成，不需要你介入。

### 2.1 自我修复

- [ ] 工具调用失败率 > 30% → 自动分析根因 → 生成修复方案 → 实施
- [ ] 上下文持续超长 → 自动调整压缩策略
- [ ] 检测到重复劳动 → 自动合并或封装

### 2.2 自扩展

- [ ] 发现重复需求模式 → 自动生成插件/命令
- [ ] 从修复记录中提炼 check 规则
- [ ] 跨会话迁移经验（"上次遇到类似问题是这样解决的"）

### 2.3 安全护栏

- [ ] 修改前自动备份
- [ ] 修改后自动跑测试
- [ ] 测试失败自动回滚
- [ ] 核心代码修改需你确认

---

## Phase 3：AIGC（创造）

> 能写代码 ≠ 能创造。创造是"这东西不存在，我想出来的"。

### 3.1 创造新工具

- [ ] 根据自然语言需求写插件（已有 `install_plugin` 能力）
- [ ] 工具自动注册、自动生成测试
- [ ] 使用效果自动反馈优化

### 3.2 创造新交互

- [ ] 发现重复模式 → 提议新命令/向导
- [ ] 自动生成交互式流程简化复杂操作

### 3.3 创造新知识

- [ ] 从项目库提炼最佳实践 → 生成文档
- [ ] 从多次修复提炼模式 → 生成 check 规则
- [ ] 跨项目迁移经验

### 3.4 开放式创新

- [ ] "你有什么建议？"— 主动提改进方案
- [ ] 识别"可以做得更好"的地方
- [ ] 探索新技术并评估是否值得接入
