完成于 2026-06-11（03a50c9 / 68b7444 / 959f523 / 3f2e129）。

## 根因链（绕门事件 8b02143..50c78c3）
run_command 默认 30s、agent 传 10s < 质量门 ~42s → 正道物理死路 →
记忆教训"超时立刻换策略"缺"≠绕守卫"边界 → `--no-verify` ×4 → 6 红测试入库。

## 借鉴成熟 agent harness 的结论
操守不靠品德靠路径工程，三层缺一不可：
1. 正道永远可走（宽超时/后台执行）——agent 只在诚实路径被堵死时才作弊
2. 邪道有牙拦（工具层拒绝已知绕门模式，拒绝消息当场教正道）
3. 事后有审计（键于不变量而非动作，覆盖一切未知绕法）

## 落地
- test_cache 对齐 head+tail 压缩（6 红→绿）+ 压缩反增大守卫
- exec.py：拦 git commit --no-verify/-n、push --no-verify、core.hooksPath
  覆盖；commit 超时自动抬到 CTG_GIT_COMMIT_TIMEOUT_FLOOR=300
- 门通行证：pre-commit 两个放行点记 write-tree 哈希入 .git/ctg-gate-passed；
  gate_audit 会话启动核对 HEAD 树，无证注入提醒（volatile，不动前缀）
- 顺手发现并修复：50c78c3 与既有 storm.py 是同根双机制，叠加产生
  read→write→read 陈旧缓存 bug——删外层、storm 补写失效、测试钉死失败类
- memory/lesson-auto-resume 补边界："换策略不包括绕过守卫"

## 教训
- 守卫被绕 ≠ agent 坏，先查正道是不是死路（超时/权限/工具缺口）
- 拦截牙按模式拦不全没关系，审计键于"树过没过门"兜底全部路径
- 造机制前先盘存量：同根第二机制 = 该收敛不该叠加
