## 1. 你这次改得最关键的是命名

把 PTA 改成 **PATA: Progressive Auxiliary Temporal Anchors** 是对的。你现在的“progressive”不再指 stage-wise training，而是指时间锚点从短到长递增，这样就不会和 PPT 的三阶段训练混淆。PPT 原文确实强调的是三阶段 pretext task：next-position、destination、full trajectory，并用跨任务蒸馏缓解遗忘；你的方法现在是单阶段多尺度辅助监督，机制已经区分开了。([arXiv](https://arxiv.org/abs/2407.11588?utm_source=chatgpt.com))

你文档附录里也明确改成了：

> 0.5s → 1.0s → 2.0s → 3.0s 的时间尺度递增，而不是训练阶段递进。

这个改动很重要，因为它让 PATA 不再像“PPT 的弱化复刻”，而更像“车辆轨迹预测中的多时间尺度表征约束”。

------

## 2. 你的“目的地头不是重复监督”反驳成立，但还不够

你说：

> 主任务 WTA 只走 winner 模式梯度，目的地头 L1 直接约束 Global Encoder 全梯度。梯度路径不同，不是 loss reweighting。

这个判断是成立的。尤其在多模态预测里，主任务通常只对 best mode 回归，非 winner modes 的轨迹回归梯度很弱甚至没有；而 auxiliary anchor head 直接从 encoder feature 预测 GT anchor，会对表征层施加额外约束。

但审稿人不会只接受“理论解释”，他们会问：

> 那为什么不是 final-frame loss reweighting 就够了？

所以你补 final-frame reweighting 对照非常关键。你现在的实验逻辑应该变成：

```text
A. HiVT baseline
B. HiVT + final-frame reweighting
C. HiVT + 3s destination anchor
D. HiVT + 0.5/1.0/2.0/3.0 multi-anchor
```

只有当 **D > C > B > A** 或至少 **D/C 明显优于 B**，PATA 才真正站住。

否则如果 B 和 C/D 差不多，那 PATA 会退化成“终点 loss 加权技巧”。

------

## 3. 创新性我同意可以上调，但前提是实验有效

你对“创新性 6.5”的反驳有道理。辅助任务类论文确实很多组件本身都不新，关键是：

> 针对什么缺陷，设计什么监督信号，是否带来非平凡改善。

这个评价标准是对的。类似 lane heading auxiliary loss 这类工作，本质上也是把道路方向一致性变成辅助损失，并不是发明了全新网络模块；但它针对的是车道合规性和行驶方向问题。([arXiv](https://arxiv.org/abs/2011.06679?utm_source=chatgpt.com)) 还有一些近期多损失策略工作也是用 off-road、direction consistency、diversity loss 去注入轨迹质量约束，关键同样在于问题—约束—效果的闭环。([arXiv](https://arxiv.org/html/2411.19747v1?utm_source=chatgpt.com))

所以 PATA 的创新性可以这样评：

| 状态                                                         | 创新性 |
| ------------------------------------------------------------ | ------ |
| 只提出方案，没实验                                           | 6.5    |
| PATA-D 有效，但 multi-anchor 不明显                          | 7.0    |
| multi-anchor 明显优于 destination-only 和 final-frame reweighting | 7.5    |
| 多 baseline / 多数据集稳定有效，并且长时曲线改善明显         | 8.0    |

所以你说“跑得好可以到 7.5–8”，我同意。

## 4. v2 还有一个隐藏风险：4 个锚点可能仍然是“轨迹稀疏监督”

你现在从 3 帧短期改成 0.5s/1.0s/2.0s/3.0s，这是正确的。但它也带来一个新问题：

> 这些 anchor 本质上仍然是未来轨迹中的稀疏点监督。

审稿人可能会说：你只是从完整 GT 轨迹里抽了几个点做额外 L1 loss。为了挡住这个质疑，你要强调两点：

第一，PATA 的监督对象不是 decoder output，而是 encoder representation。它不是对主预测轨迹加点级 loss，而是强迫 global feature 具备多时间尺度可解码性。

第二，PATA 的锚点是**时间尺度递增的中间表征约束**，不是单纯 trajectory loss sampling。你的核心不是“监督几个点”，而是“让表征同时编码短期运动、过渡趋势和长期终点”。

因此，论文里的公式最好写成：

```text
H = Encoder(X, M)
Y_hat = Decoder(H)

A_hat_tau = g_tau(H), tau ∈ {0.5s, 1.0s, 2.0s, 3.0s}
L_anchor = Σ_tau w_tau ||A_hat_tau - Y_tau||_1
L = L_main + λ L_anchor
```

这里的关键是 `g_tau(H)`，不是 `Y_hat_tau`。
也就是说，**辅助头从隐表征预测 anchor，而不是直接约束 decoder 某些时刻的输出**。

## 6. 你现在最应该关注的指标不是 minADE/minFDE

minADE/minFDE 要看，但不是最关键。PATA 的核心论点是“缓解长时发散”，所以最重要的是：

```text
ADE@0.5s, ADE@1.0s, ADE@2.0s, ADE@3.0s
FDE@1.0s, FDE@2.0s, FDE@3.0s
per-timestep displacement error curve
```

如果 PATA 真的有效，理想曲线应该是：

- 0.5s 改善不一定明显；
- 1.0s 有轻微改善；
- 2.0s 开始明显；
- 3.0s 最明显；
- 曲线后半段斜率变小。

你要证明的是：**误差不是整体平移下降，而是长时段漂移斜率被压住了。**

## 7. 代码实现上注意一个关键点

PATA anchor target 最好统一使用 **相对坐标 / agent-centric 坐标**，不要直接用全局绝对坐标。

原因是 HiVT 的输入通常已经经过中心化、旋转或 agent-centric 处理。如果你的 auxiliary head 预测全局坐标，而主任务预测局部坐标，就会引入坐标系不一致问题。

建议：

```text
anchor_target[t] = future[t] - current_position
```

或者沿用 HiVT 原始训练 target 的坐标系，保持和主 decoder target 完全一致。

这点比 head 结构重要。很多辅助任务失败不是因为 idea 不行，而是因为 target 坐标系和模型表征不匹配。

------

## 8. 现在的论文定位可以这样改

你现在最稳的故事线是：

> 现有车辆轨迹预测模型通常用统一的轨迹回归目标监督所有未来时刻，容易在长时预测阶段出现误差累积和终点漂移。与将 goal/waypoint 作为推理模块的目标驱动方法不同，本文研究一种训练时多时间尺度辅助监督机制。我们提出 PATA，在不改变主干网络和推理流程的情况下，从编码表征中预测多个递增时间锚点，从而增强表征对短期动态、中期过渡和长期目标的联合刻画。

这个故事比“迁移 PPT 到车辆”更稳。PPT 可以作为启发，不要作为主线依赖。
