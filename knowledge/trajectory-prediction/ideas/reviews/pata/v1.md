------

## 1. 总体结论

你的设计核心是：受 PPT 行人轨迹预测的渐进式预文本任务启发，把“短期动态、长期目的地、驾驶意图”转化为车辆轨迹预测中的辅助监督信号，并挂到 HiVT 上训练。你文档里明确把方案 B 定位为“不改训练范式，只加 loss 头”，即主任务外增加短期位移、目的地、意图分类三个辅助头，这一点是合理的。

我的判断是：

**可做性：高。**
代码改动小，监督信号可以从 GT 轨迹直接构造，不依赖额外人工标注，适合你现在单人 + A40 的资源条件。

**论文潜力：中等偏上。**
如果只是“HiVT + 3 个辅助头”，创新性不算强；但如果你能证明它确实缓解了长时预测漂移，并且在多个 baseline 或多个长时指标上稳定有效，就可以成为一篇不错的辅助学习 / 训练策略型论文。

**最大风险：不是代码，而是故事是否站得住。**
你现在把它叫“渐进式时间辅助任务”，但实际推荐方案是单阶段多任务 loss，并不是真正的 progressive training。这个命名和机制之间有一点不一致。

------

## 2. 最值得肯定的地方

第一，**问题切入点是对的**。你没有再去堆一个新 backbone，而是抓住“长时预测发散 / FDE 高 / 终点漂移”这个具体问题。这比泛泛地说“提高轨迹预测精度”更像一个可发表的科研问题。

第二，**辅助任务设计和轨迹预测主任务有天然联系**。短期位移对应局部运动趋势，目的地对应长期锚点，意图分类对应离散驾驶模式。文档中对三个头的位置设计也基本合理：短期头接 local feature，目的地和意图接 global feature。

第三，**你已经提前考虑了与 DenseTNT/TNT 的区别**。这很重要。你把 PTA 和 goal-based 架构区分为“训练时辅助信号”和“推理时必经模块”，这个区分是成立的，也能用于审稿 rebuttal。

第四，**实验路线比较清晰**。Baseline、仅短期、仅目的地、仅意图、组合版本、HiVT-128 上限参考，这套消融是能支撑论文的。文档里也把 minADE、minFDE、off-road rate、意图分类准确率作为观察指标，这个方向是对的。

------

## 3. 主要问题：现在的“新颖性判断”有些过满

你文档中多次写到“完全空白”“车辆方向没人做过”“第一个将时间渐进式辅助任务用于车辆轨迹预测”。这些表述要谨慎。即使真的没有完全一样的工作，审稿人也可能从以下方向挑战你：

1. goal-based trajectory prediction；
2. waypoint-based prediction；
3. intention-conditioned prediction；
4. multi-task learning for motion forecasting；
5. auxiliary loss / self-supervised trajectory representation learning；
6. curriculum learning / progressive learning in forecasting or planning。

所以更稳的说法不是：

> 车辆轨迹预测领域完全空白。

而是：

> 现有车辆轨迹预测方法多将目标点、意图或 waypoint 作为架构推理流程的一部分，而较少系统研究这些时间尺度相关信号作为训练时辅助监督，对长时预测漂移的影响。

这个表述更安全，也更专业。

------

## 4. 最大技术隐患：目的地辅助头可能“重复监督”

你的主任务本来就通过 FDE / WTA regression 监督整条轨迹，最终帧已经在主损失里。因此，单独加一个 destination head，本质上可能被审稿人质疑为：

> 你只是把最后一帧监督又加了一遍，为什么这不是 loss re-weighting？

这是你必须处理的核心问题。

要让目的地头成立，不能只说“终点是长期锚点”。你需要证明它提供了**不同的梯度路径**：

- 主 decoder 的 FDE 监督只作用在 winner mode 上；
- destination head 直接约束 encoder representation；
- 它不参与推理，只作为 representation shaping；
- 它能让 global feature 更显式编码长期意图。

所以论文里应强调：**PTA 不是简单加大最终帧 loss，而是通过 auxiliary head 对中间表征施加长期目标约束。**

实验上你必须加一个对照：

**Baseline + final-step loss reweighting**

也就是不加 destination head，只把主轨迹 loss 中最后一帧权重加大。
如果 PTA-D 比这个还好，你的目的地辅助头才真正有说服力。

------

## 5. 第二个隐患：短期头可能没什么用

HiVT 这类模型短期预测通常已经比较强，尤其 Argoverse 1 的 3 秒预测里，前 0.3s/0.5s 基本由当前速度和局部历史决定。你文档里也意识到“短期头提升不显著”的风险。

我建议你不要一开始就把短期头作为核心贡献。它更适合作为辅助消融项，而不是主卖点。

更好的设计是把短期头改成**多尺度中短期锚点**：

- 0.5s；
- 1.0s；
- 2.0s；
- 3.0s destination。

这样比只预测未来 3 帧更像“temporal hierarchy”。否则 0.1s/0.2s/0.3s 太短，审稿人会觉得它只是速度估计。

------

## 6. 第三个隐患：意图标签规则比较脆

你设计的意图分类包括直行、左转、右转、左变道、右变道，并通过 heading change、lateral displacement、lane change 自动标注。这个想法可行，但实现上风险最大。

问题在于：

1. Argoverse 轨迹只有 3 秒，有些转弯或变道还没完成；
2. 最近车道匹配可能不稳定；
3. intersection 内的 lane id 切换不一定等于变道；
4. 左转 / 左变道、右转 / 右变道容易混淆；
5. 非 ego agent 的地图关联质量不一定都可靠。

所以意图头不能直接作为强贡献。它应该是第三阶段扩展项。你必须先统计标签分布，并抽样可视化 100–200 个 case。文档里提到“人工检查 100 个样本”是正确的，但我建议把它提升为正式实验的一部分：报告 auto-label agreement 或至少给出可视化案例。

------

## 7. 命名建议：PTA 这个名字可以保留，但要改解释

现在最大的问题是：你说它来自 Progressive Pretext Task Learning，但你推荐方案其实是单阶段多任务训练。严格来说，它不是 progressive training。

建议你把 PTA 解释为：

> Progressive Temporal Auxiliary Supervision

而不是强调 progressive training。

也就是说，“progressive”指的是**时间尺度上的渐进监督**，不是训练过程上的 stage-wise progressive learning。

你可以把三个任务重命名为：

- short-horizon auxiliary；
- mid-horizon waypoint auxiliary；
- long-horizon destination auxiliary；
- optional intent auxiliary。

这样故事会更严谨。

------

## 8. 我建议你实际先做的最小版本

不要直接做三头全开。按这个顺序：

### 第一步：PTA-Destination

只加目的地头。

核心验证：

- dest_loss 是否下降；
- minFDE@6 是否下降；
- minADE@6 是否不明显下降但不恶化；
- 与 final-frame loss reweighting 对比；
- 与 HiVT-64 baseline 对比。

如果这个版本没有效果，后面三个头大概率也撑不起论文。

### 第二步：PTA-MultiHorizon

不要只做 3 帧短期预测。改成预测多个关键时间点：

```text
future[5], future[10], future[20], future[29]
```

对应 0.5s、1.0s、2.0s、3.0s。

这个比“短期 + 目的地”更自然，也更像时间层级辅助任务。

### 第三步：Intent 作为可选增强

意图头只有在标签质量可靠、且能提升 mode coverage / miss rate 时再加。否则它可能拖累主线。

------

## 9. 必须补的实验

你现在的实验设计还缺几个关键对照：

| 必补实验                                    | 为什么重要                                     |
| ------------------------------------------- | ---------------------------------------------- |
| final-frame loss reweighting                | 排除“只是重复加强终点监督”的质疑               |
| multi-horizon auxiliary vs destination-only | 证明“渐进式时间尺度”确实比单终点强             |
| aux head train-only vs inference-used       | 证明零推理开销定位成立                         |
| 不同 λ 敏感性                               | 防止被认为靠调参碰出来                         |
| 失败案例分析                                | 说明在哪些场景有效：转弯、交叉口、长直道、变道 |
| per-timestep ADE 曲线                       | 直接证明长时漂移缓解，而不是整体偶然下降       |

尤其是 **per-timestep ADE/FDE curve** 很关键。你的主题是长时发散，就不能只报 minADE/minFDE。要画 1s、2s、3s 的误差曲线。

## 回复

### 同意并已改

| 评审意见 | 修改 |
|:--------|:-----|
| "渐进式"名不副实 | PTA→PATA，"渐进"→时间尺度递增，非训练阶段 |
| 短期3帧(0.3s)太短 | →多尺度锚点 0.5s/1.0s/2.0s/3.0s |

| 缺final-frame reweighting对照 | →已补入实验设计 |
| 缺per-timestep ADE曲线 | →已补入实验设计 |

### 反驳

**对"目的地头=重复监督"的反驳**：主任务WTA只走winner模式梯度，目的地头L1直接约束Global Encoder全梯度。**梯度路径不同，不是loss reweighting。** 同意实验上需要final-frame reweighting对照来证明这一点。

**对"创新性6.5"的抬杠**：几乎所有辅助任务论文的组件都不新——SSL-Int的4个pretext、PPT的next-pos+destination、CombAux的offroad+direction+diversity哪个是新的？新在于"什么组合→针对什么问题→是否产生非平凡效果"。但分数接受的前提是实验还没跑，跑得好可以到7.5-8。
