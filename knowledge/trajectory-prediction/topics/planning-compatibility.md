# 子方向：规划兼容性（Planning Compatibility）

> **核心问题**：轨迹预测模型用 minADE/minFDE/MR 优化，但这些指标和下游规划器实际需求脱节。
> **空白等级**：⭐⭐⭐⭐⭐（当前所有子方向中最空的）
> **卡片数**：0 篇专门论文，只有间接相关

## 问题陈述

预测模型对"好预测"的定义和规划器不同：

```
预测指标            规划器视角
minADE↓            "轨迹接近真值但不一定安全"
minFDE↓            "终点准了但路径上可能撞了"
MR↓                "主模态选了最安全的但不是最准的"
```

例：
- 预测轨迹有 0.5m ADE 误差，但横穿了实线 → **规划器用不了**
- 另一条预测误差 1.2m，但完全在车道内 → **规划器可以用**
- 现有指标偏好前者（ADE 更低），但规划器偏好后者

## 间接相关的已有工作

| 论文 | 做了什么 | 和规划兼容性的关系 |
|---|---|---|
| CCTR (AAAI 2024) | 校准不确定性→提升规划器安全 | 证明"校准预测→规划受益"链路可行 |
| TPK (arxiv 2025) | 运动学约束→轨迹物理可行 | 证明"约束预测→更可信"链路可行 |
| Plan-MAE (arxiv 2025) | 自监督预训练→预测+规划 | 但两阶段，不是训练时辅助loss |
| SeerDrive (NeurIPS 2025) | 双向建模场景演化+规划 | 端到端，不专攻预测 |
| UniTraj (ECCV 2024) | 发现跨域泛化差 | 间接说明预测模型"不够鲁棒" |

## 关键待解决的问题

1. **量化**：规划兼容性应该用什么指标衡量？（碰撞率？规划器成功率？安全距离裕度？）
2. **标签**：如何从数据中自动生成"规划兼容"的监督信号？（无需人工标注）
3. **可微性**：规划兼容性损失函数是否可微？
4. **消融**：规划兼容性辅助任务是否独立于已有辅助任务（校准、物理约束）带来额外收益？

## 可参考方向的思路

1. 从数据中提取"预测轨迹是否与规划冲突"的标签（用规则/地图检查）
2. 设计可微碰撞距离函数作为辅助损失
3. 用规划器的后验反馈（这个预测好用吗？）作为训练信号
