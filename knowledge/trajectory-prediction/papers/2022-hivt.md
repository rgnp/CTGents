# HiVT: Hierarchical Vector Transformer for Multi-Agent Motion Prediction

| 字段 | 内容 |
|---|---|
| **会议** | CVPR 2022 |
| **作者** | Zikang Zhou, Luyao Ye, Jianping Wang, Kui Wu, Kejie Lu |
| **代码** | [GitHub](https://github.com/ZikangZhou/HiVT) |
| **任务** | 多智能体轨迹预测（multi-agent motion prediction） |
| **数据集** | Argoverse 1.1 |

---

## 核心贡献（一句话）

提出分层式向量 Transformer 架构，将问题分解为**局部上下文提取** + **全局交互建模**，在保证精度的同时实现实时推理。

---

## 架构设计

```
输入（地图向量 + 历史轨迹）
  ↓
┌─ Local Encoder（per-agent）────┐
│  map features → map polyline FEt│
│  agent history → agent FE       │
│  self-attention → local context  │
└────────────────────────────────┘
  ↓
┌─ Global Encoder（scene-level）──┐
│  cross-attention (agent↔map)    │
│  self-attention (agent↔agent)   │
│  → 交互感知的场景上下文          │
└────────────────────────────────┘
  ↓
┌─ Decoder────────────────────────┐
│  trajectory proposals → refine  │
│  输出 K 条多模态轨迹             │
└────────────────────────────────┘
```

**关键设计决策**：
1. **分层解耦** — 每个 agent 先独立编码局部上下文，再聚合全局交互，减少计算复杂度
2. **平移不变性** — scene representation 以目标 agent 为中心，不依赖全局坐标系
3. **旋转不变性** — 空间位置编码使用相对方位而非绝对坐标
4. **双变体** — HiVT-64 (embed_dim=64, 速度优先) / HiVT-128 (embed_dim=128, 精度优先)

---

## 辅助任务设计

HiVT 本身不使用辅助学习目标。训练损失为**负对数似然（NLL）+ 回归损失**的标准组合，输出 Gaussian Mixture Model 参数。

这意味着：**HiVT 是"裸架构"，没有辅助任务。** 所有辅助学习目标类的方法都可以在其之上叠加，这是本项目最合适的基线模型。

---

## 实验结论

| 模型 | minADE@6 | minFDE@6 | 参数量 | 推理速度 |
|---|---|---|---|---|
| HiVT-64 | 0.80 | 1.17 | 0.69M | **最快** |
| HiVT-128 | 0.74 | 1.11 | 2.83M | 快 |
| LaneGCN | 0.87 | 1.36 | 3.0M | 慢 |
| SceneTransformer | 0.80 | 1.23 | 20.0M | 慢 |

- HiVT-64 以 0.69M 参数达到与 SceneTransformer 可比的精度，速度快一个数量级
- 统一预测所有 agent，不逐个遍历

---

## 后续影响（这条线的价值）

HiVT 是 Argoverse 1.1 上的经典基线，后来被大量工作作为：
- **编码器骨干**：许多方法直接使用 HiVT 的 local encoder 提取特征
- **对比基线**：几乎所有 2022-2024 的 Argoverse 论文都会对比 HiVT
- **轻量化参考**：证明"少参数也能做好"的设计哲学
