# Markdown 报告生成 Skill

## 概述
按照专业标准生成结构清晰、排版美观的 Markdown 报告。

## 步骤

### 1. 理解需求
- 确认报告类型（周报/技术文档/项目总结/研究笔记）
- 确认目标读者（技术团队/管理层/公开）

### 2. 组织结构
使用标准的 Markdown 层级：
- `#` 主标题（仅一个）
- `##` 一级分节
- `###` 二级分节
- `####` 三级分节（避免过多嵌套）

### 3. 内容规范
- 使用 `- ` 无序列表和 `1. ` 有序列表
- 代码块标注语言：```python
- 表格使用 GFM 格式
- 重要内容用 **加粗** 或 `行内代码`

### 4. 格式要求
- 段落之间空一行
- 列表项之间不空行
- 引用用 `> `
- 分隔线用 `---`
- 链接用 `[文字](url)`

### 5. 输出检查
- ✅ 层级不超过 4 层
- ✅ 没有未闭合的代码块
- ✅ 表格对齐正确
- ✅ URL 格式正确
