# 🧪 Agent Skill 蒸馏设计方法论

> 从同事.skill 和女娲.skill 中学到的设计模式

## 一、什么是"蒸馏"？

蒸馏不是复制，而是**提取精髓、重构表达、迁移创造**的过程。

```
原始灵感 → 解构分析 → 提取模式 → 跨领域迁移 → 原创重构
```

---

## 二、蒸馏出的五大设计模式

### 模式 1️⃣ 角色锚定（Persona Anchoring）

**学到的：**
- 同事.skill → 用一个日常身份建立亲和力
- 女娲.skill → 用一个神话原型赋予使命感

**模式公式：** `一个鲜明的角色 + 一句核心宣言 + 一套价值观`

**作用：** 角色是整个 Skill 的灵魂，决定了 Agent 输出的语气、态度和思维框架。

### 模式 2️⃣ 哲学先行（Philosophy First）

**学到的：**
- 同事.skill 的"像同事一样说话、先理解再行动、给出理由"
- 女娲.skill 的"少即是多、防御性编程、可读性优先"

**模式公式：** `3-5 条核心原则，每条一句，构成决策的底层逻辑`

**作用：** 原则比步骤更重要——原则让 Agent 在没遇到过的情况中也能做出正确判断。

### 模式 3️⃣ 多技能模块（Multi-Skill Modules）

**学到的：**
- 同事.skill 拆成：Code Review / 技术方案 / Bug 分析 / 重构建议
- 女娲.skill 拆成：代码生成 / 重构 / 补全 / 测试 / 文档

**模式公式：** `将一个大领域拆成 3-6 个原子技能，每个有独立的触发条件和流程`

**作用：** 模块化让 Agent 知道"什么时候该做什么"，避免万能药式的模糊回答。

### 模式 4️⃣ 流程剧本化（Scripted Workflow）

**学到的：**
- 每个子技能都有明确的"步骤编号 1→2→3→4"
- 不是自由发挥，而是有剧本的即兴表演

**模式公式：** `步骤编号 + 动词开头 + 每步有检查点`

**作用：** 步骤化降低了 Agent 的决策负担，保证输出的一致性和质量下限。

### 模式 5️⃣ 边界清单（Edge Case Inventory）

**学到的：**
- 两个 Skill 都有专门的"边界处理"章节
- 用表格或列表明确列出异常场景的处理方式

**模式公式：** `表格列出 5-8 种异常场景 + 对应的处理策略`

**作用：** 边界清单防止 Agent 在遇到特殊情况时"懵掉"或胡编。

## 四、跨领域迁移示例

| 设计模式 | 代码领域 | 产品领域 | 写作领域 |
|---------|---------|---------|---------|
| 角色锚定 | 资深同事 / 女娲 | 产品教练 | 写作导师 |
| 哲学先行 | 少即是多 | 用户至上 | 清晰即优雅 |
| 多技能模块 | 审查/重构/测试 | 需求/原型/验收 | 结构/润色/风格 |
| 流程剧本化 | 1理解→2分类→3反馈 | 1调研→2分析→3输出 | 1审读→2修改→3验证 |
| 边界清单 | 大代码/不确定 | 需求模糊/冲突 | 风格不统一/过长 |

---

> 💡 蒸馏的本质：**找到结构，而不是复制内容。**
